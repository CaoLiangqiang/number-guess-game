# number-guess-game 开发计划

## 项目概述

**数字对决 Pro** 是一款基于 H5 的数字推理对战游戏，支持：
- 单机人机对战（AI 使用 Minimax + 信息熵算法）
- 双人实时联机对战（WebSocket）
- PWA 支持（可安装到主屏幕，离线游玩）

**当前版本**: 前端 v2.1.0 | 服务端 v1.0.0

---

## 发现的问题

### 1. 版本号不一致
- **类型**: 待修复
- **位置**: `package.json:3` (v2.1.0), `server/package.json:3` (v1.0.0)
- **描述**: 前端版本为 2.1.0，服务端版本为 1.0.0，两者版本不同步
- **优先级**: 中

### 2. 缺少 CHANGELOG
- **类型**: TODO
- **位置**: 项目根目录
- **描述**: 项目缺少 CHANGELOG 文件，无法追踪版本变更历史
- **优先级**: 中

### 3. 设计文档过时
- **类型**: TODO/未完成
- **位置**: `design_doc.md`
- **描述**: 设计文档内容已过时，包含大量未实现的架构设计（根据 .trae/specs/pwa-conversion/spec.md 指出）
- **优先级**: 低

### 4. 联机服务器未实际部署
- **类型**: BUG/未完成
- **位置**: `server/server.js`
- **描述**: 联机模式的服务器代码已实现，但游戏中没有配置实际可访问的服务器地址，用户无法真正进行联机对战
- **优先级**: 高

### 5. Redis 依赖未配置
- **类型**: 待修复
- **位置**: `server/package.json` (依赖 redis)
- **描述**: 服务器依赖 Redis 但生产环境可能未配置，当前使用内存存储作为降级方案，缺少明确文档说明
- **优先级**: 中

### 6. 调试日志未清理
- **类型**: TODO
- **位置**: `js/network.js:7-8`, `index.html` 多处 console.log
- **描述**: 代码中存在多处 console.log 调试语句，虽然已添加 DEBUG 开关但生产环境仍可能有输出
- **优先级**: 低

### 7. 缺少生产环境部署配置
- **类型**: TODO
- **位置**: 项目根目录
- **描述**: 缺少 Docker、Kubernetes 或其他生产环境部署配置文件
- **优先级**: 中

### 8. 缺少 CI/CD 流程
- **类型**: TODO
- **位置**: `.github/` 目录
- **描述**: 虽然存在 .github 目录，但未配置完整的 GitHub Actions CI/CD 流程（自动测试、构建、部署）
- **优先级**: 中

### 9. 文档中的冗余文件
- **类型**: TODO
- **位置**: `docs/DEPLOY_GUIDE.md`
- **描述**: 根据 PWA spec 说明，某些文档需要清理或整合
- **优先级**: 低

### 10. WebSocket 安全考虑
- **类型**: 待修复
- **位置**: `server/server.js`
- **描述**: 服务器缺少 WebSocket 连接速率限制、心跳检测、超时断连等保护机制
- **优先级**: 中

### 11. 代码覆盖率不足
- **类型**: TODO
- **位置**: `tests/` 目录
- **描述**: 现有测试主要覆盖核心游戏逻辑和网络模块，但 UI 测试、集成测试覆盖不足
- **优先级**: 中

### 12. PWA 安装提示 UI 可优化
- **类型**: 优化
- **位置**: `index.html:2901-2903`
- **描述**: PWA 安装提示 UI 较为简单，可增加更友好的交互
- **优先级**: 低

## 建议的开发计划

### 第一阶段（紧急）- 完善核心功能

- [ ] **修复版本号一致性**: 统一前端和服务端版本号
- [ ] **配置联机服务器**: 在游戏中添加可配置的服务器地址，或提供默认演示服务器
- [ ] **添加 CHANGELOG**: 创建 CHANGELOG.md 记录版本变更历史
- [ ] **清理调试日志**: 在生产环境构建时移除或禁用 console.log

### 第二阶段（重要）- 完善基础设施

- [ ] **添加生产部署配置**: 创建 Docker Compose 或 Kubernetes 配置
- [ ] **完善 Redis 配置**: 明确 Redis 配置要求，添加配置文档
- [ ] **增强 WebSocket 安全**: 添加速率限制、心跳检测、异常连接处理
- [ ] **添加 CI/CD 流程**: 配置 GitHub Actions 自动测试和部署
- [ ] **增加测试覆盖率**: 添加更多单元测试和 E2E 测试用例

### 第三阶段（优化）- 完善体验

- [ ] **更新设计文档**: 清理过时的 design_doc.md 或更新为精简版
- [ ] **优化 PWA 安装体验**: 改进安装提示 UI，增加用户引导
- [ ] **添加更多动画效果**: 提升游戏体验
- [ ] **支持排行榜功能**: 增加玩家排行榜（需要服务器支持）
- [ ] **支持更多游戏模式**: 如限时模式、回合限制模式等

## 总结

该项目结构良好，代码质量较高，核心游戏逻辑完整。首要任务是：
1. 解决版本号不一致问题
2. 配置可用的联机服务器
